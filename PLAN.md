## 1. Требования

### Функциональные
- Один зашитый сценарий: семья Айдар/Алия/Айша(9)/Тимур(5), маршрут Алматы→Астана, **ночной поезд (~13 ч)**, **5–7 июня** (отправление чт 4 июня вечер ~20:00, прибытие пт 5 июня утро ~09:00).
- Клиент проходит путь поездки в чате: фаза 0 (Т−14) → фаза 1 (Т−3) → фаза 2 (в поезде) → завершение.
- На каждом шаге AI генерирует короткое сообщение «языком тревоги»; клиент отвечает чипсами (HITL).
- Справа — живой Travel Plan: 10 пунктов (locked/wait/done), тикет, прогресс фаз, бюджет (факт+расчётное) в реальном времени.
- Кнопки симуляции «▶ за 3 дня» и «▶ в поезде» двигают сценарий между фазами.
- Состояние сохраняется (план переживает рестарт контейнера во время демо).

### Нефункциональные (для демо)
- Запуск: `docker compose up` → `localhost:8000`, end-to-end за один шаг.
- **Без сети демо всё равно проходит целиком** — на каждый AI-шаг есть заготовленный `fallback`.
- Один пользователь, один прогон за раз. Нагрузки/масштаб не рассматриваем.

### Ограничения
- Бэкенд Django, фронт React, БД SQLite, всё локально в Docker Desktop.
- LangChain / LangGraph — обязательное ядро оркестрации (заменяет самописный раннер из прошлой версии).
- Вызов Модели LLM через API: `claude-haiku-4-5`, ключ в `.env`.
